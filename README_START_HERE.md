# PriceBot SQLite Edition

هذه النسخة تستخدم SQLite لتخزين:

- المنتجات
- الطلبات
- ذاكرة الأسئلة والكاش
- ذاكرة الصور
- تصحيحات الأدمن
- عداد استخدام AI اليومي

## مهم

لا ترفع هذه الملفات إلى GitHub:

- `.env`
- `pricebot.db`
- `*.db-wal`
- `*.db-shm`
- `venv/`
- `__pycache__/`

## ماذا يحدث عند أول تشغيل؟

إذا لم توجد قاعدة بيانات، سيقوم البوت تلقائيًا بإنشاء:

`/opt/pricebot/pricebot.db`

ثم يستورد تلقائيًا من الملفات القديمة إن وجدت:

- `products.csv`
- `orders.csv`
- `memory.json`

بعدها يكون التشغيل الأساسي من SQLite، مع بقاء CSV/JSON كنسخ احتياطية ومرايا للتصدير.

## روابط مهمة

- المنتجات: `/admin?key=PriceBotAdmin2026`
- الطلبات: `/admin/orders?key=PriceBotAdmin2026`
- الذاكرة: `/admin/memory?key=PriceBotAdmin2026`
- قاعدة البيانات: `/admin/database?key=PriceBotAdmin2026`



## تحديث حل المشاكل التشغيلية
- الطلبات الآن لا تعتبر مؤكدة تلقائياً. الزبون يستلم: الطلب بانتظار تأكيد الصيدلية.
- لوحة الطلبات تحتوي أزرار: تأكيد للزبون / رفض / تم التنفيذ.
- تقرير يومي تلقائي للأدمن يومياً 21:00 عبر cron، مع زر إرسال يدوي من لوحة الطلبات.
- البحث تحسن للأسماء المركبة مثل: بروفين 400، بنادول اكسترا.
- سؤال "شنو عندكم؟" يعرض عينة من المنتجات المتوفرة.
- تم إزالة تكرار page_layout.
- الذاكرة والكاش تستخدم SQLite مباشرة بدل فتح وكتابة memory.json مع كل رسالة.


## تحديث الإحصائيات الاحترافية
- قاعدة SQLite تسجل استعلامات المنتجات في جدول `product_inquiries`.
- التقرير اليومي يرسل للأدمن أكثر المنتجات سؤالاً: اليوم/الأسبوع/الشهر.
- صفحة الإحصائيات: `/admin/analytics?key=PriceBotAdmin2026`.
- يتم تسجيل الاستعلامات بدون طباعة مفاتيح أو أسرار.


## لوحة التاجر

تمت إضافة لوحة تاجر منفصلة وآمنة عن لوحة الأدمن الكاملة.

الرابط:

```text
/merchant/login
```

الرمز الافتراضي:

```text
BADR2026
```

يمكن تغيير الرمز من لوحة الأدمن في الإعدادات عبر الحقل `MERCHANT_CODE`.

لوحة التاجر تحتوي على:
- الرئيسية والملخص السريع
- الطلبات وتأكيد/رفض/تنفيذ الطلب
- المنتجات وتعديل السعر والتوفر والملاحظات
- رفع ملف Excel/CSV
- الإحصائيات
- إعدادات المحل ورقم تنبيهات الأدمن
- رابط واتساب و QR جاهز للنشر للزبائن

لا ترفع `.env` أو `pricebot.db` أو `venv` إلى GitHub.
