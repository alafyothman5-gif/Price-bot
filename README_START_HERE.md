# PriceBot — صيدلية بدر البشرية + Multi AI Fallback

هذه نسخة جاهزة لـ GitHub والتشغيل على السيرفر.

## الموجود في النسخة

- ردود مخصصة باسم: صيدلية بدر البشرية.
- المدينة: أجدابيا.
- ساعات العمل: 24 ساعة.
- التوصيل غير متوفر حالياً.
- لوحة منتجات مناسبة للهاتف.
- لوحة طلبات مناسبة للهاتف.
- رفع CSV / Excel.
- دعم أسماء بديلة للمنتجات.
- ذكاء محلي مجاني للبحث الأساسي.
- AI API حقيقي مع أكثر من مفتاح وأكثر من مزود.

## ترتيب الذكاء الاصطناعي

البوت يحاول المفاتيح بهذا الترتيب الافتراضي:

```text
OpenRouter → Gemini → Groq → OpenAI
```

إذا فشل مفتاح أو انتهى الحد اليومي أو ظهرت مشكلة في مزود، ينتقل تلقائياً للمفتاح التالي، ثم المزود التالي.

يمكن تغيير الترتيب من الإعدادات:

```text
AI_PROVIDER_ORDER=openrouter,gemini,groq,openai
```

## التشغيل على السيرفر

بعد رفع الملفات إلى GitHub، استخدم سكربت النشر الموجود لديك أو شغل:

```bash
cd /opt/pricebot && bash DEPLOY_ON_SERVER.sh
```

السكريبت يحافظ على:

- `.env`
- `products.csv`
- `orders.csv`

ولا يضع أي أسرار داخل GitHub.

## تفعيل Multi AI من اللوحة

افتح:

```text
/admin/settings?key=PriceBotAdmin2026
```

ثم:

1. فعّل خيار: تفعيل AI API.
2. اترك ترتيب المحاولة كما هو أو عدله.
3. ضع موديلات المزودين أو اترك الافتراضي.
4. الصق المفاتيح في خاناتها:
   - OpenRouter keys
   - Gemini keys
   - Groq keys
   - OpenAI keys
5. يمكن وضع أكثر من مفتاح، كل مفتاح في سطر.
6. اضغط حفظ.
7. اضغط اختبار AI.

لا ترسل مفاتيح API في الدردشة ولا ترفعها إلى GitHub.

## كيف يعمل AI داخل البوت

البوت لا يفتح دردشة عشوائية. AI يستخدم فقط لفهم نية الزبون مثل:

- كم سعر البروفين؟
- عندكم بندول؟
- نبي شي للصداع
- في أوجمنتين؟

ثم يبحث في قائمة المنتجات الموجودة فقط. لا يخترع منتجاً غير موجود.

لو الزبون سأل عن جرعة أو استعمال أو حالة طبية، البوت لا يعطي نصيحة علاجية، بل يطلب مراجعة الصيدلي.
